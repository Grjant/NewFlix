Data Sources:

SerpAPI

to scrape Google Search results for a theater name. If a successful scraping occurs, SerpAPI returns a results json with data on showtimes that includes days, times, type of movie, and the movie name for that theater. This API allowed us to get the attributes for our theaters instances. In the future, we want to make our website more flexible and use another API, MovieGlu, to find the theaters that are nearby, so users do not have to manually type in a theater name.

OMDbAPI

We started off by manually making calls to the API with a sample of ten movies. Eventually, we decided to add more movies by using the movies that were currently showing at the theaters in our database. As theaters are being added to the database, our program will check if the movies they are showing exist in the database. If the movie exists in the database, it will establish the link between the showing and the movie. If it does not exist, our program will call OMDbAPI and add the movie to the database and connect it to the movie showing.

RapidAPI Streaming Availability

Since we needed to link streaming services and movies, we needed an API that could return whether a movie was contained in a service’s catalog. After entering the movies into the database, we query the database to get all of the movies and run each of them through RapidAPI to see what services they are available on and establish the link in the database.
